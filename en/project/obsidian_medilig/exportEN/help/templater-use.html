Templater Use

Templater is a template language that lets you insert variables and functions results into your notes. It will also let you execute JavaScript code manipulating those variables and functions. With Templater, you will be able to create powerful templates to automate manual tasks.

Commands and Functions

To understand how Templater works, let's define a few terms:

  • A template is a file that contains commands.
  • A text snippet that starts with an opening alligator bracket <%, ends with a closing alligator bracket %> is what we will call a command.
  • A function is an object that we can invoke inside a command and that returns a value (the replacement string)

There are two types of functions you can use:

  • Internal functions. They are predefined functions that are built within the plugin. As an example, tp.date.now is an internal function that will return the current date.
  • User functions. Users can define their own functions. They are either system command or user scripts.

Command Types

Templater defines 2 types of opening alligator brackets, that defines 2 types of commands:

Interpolation command

<% %>
Interpolation command. It will output the result of the expression that's inside.

Javascript Execution Command

<%* %>
This type of command allows us to execute JavaScript code. With a JavaScript Execution command, we can pretty much do everything that JavaScript allows us to do. Some examples are given below. We can still access the tp object and all the internal variables / functions from this type of command. JavaScript Execution commands let you access global namespace variables. This means you can access things like app or moment.

Asynchronous commands

Some internal functions are asynchronous. When calling such functions inside a JavaScript execution command, don't forget to use the await keyword if necessary.
It is important to note that the tp.system.prompt() and tp.system.suggester() both require a await statement to save the value to a variable

Sometimes, you may want to output something when using a JS execution command. When our templating engine generates a replacement string using all of our commands results, it is stored in a variable named tR. This is the string that will contain the processed file content. You are allowed to access that variable from a JS execution command. This means that, to output something from a JS execution command, you just need to append what you want to output to that tR string variable.

For example, the following command: <%* tR += "test" %> will output test.

How to work with Templater

Default location for new notes

First we have to go at Settings/Files and links and change the option for the default location for new notes.

We want newly created notes to be placed at the same folder as currently open file

Shortcuts

Replace templates in the active file (Alt + E)

This mode is normally used for testing template language code.


It replaces the code in the active file by pressing e.g. Alt+E (execute template language)


and then we can return back to the previous state of the file with the template code by pressing Ctrl + Z (undo command)

Open Insert Template modal (Alt + I)

In this mode we select a template that will be inserted at the cursor point inside an Obsidian note (markdown file) that is already open

How to avoid creating untitled notes when triggering the creation of a new file
Extra blank “Untitled” note created when triggering new template creation

This can occur when you create a new note inside a folder based on a specific template or you apply the Create new note from template command and inside the template code you call the function tp.file.create_new

One way to avoid this pitfall is to invoke templater with the templater icon or use the Open Insert Template modal command (Alt + I)

You can also define a hotkey to execute the specific template

Create new note from template (Alt + N)

The recommended way to create new note from template is to right click on the folder that we want to create the new note and select the command Create new note from template

Executing this command it will open a pop-up menu with all the templates we have previously defined in the Template folder location setting of Templater plugin. When we select that template, it will try to apply it on the new file that will be created in the specified folder

Another way is to press the shortcut Alt+N when the cursor is on an active note that has been already opened. When we select the template to apply a new note will be created in the same folder with the active note.

New Note within a folder based on a specific template

We have already seen how to create a new note with Templater plugin by pressing Alt+N. Here we learn another way by creating a New note within a folder. Here we have already configured the Templater plugin with a specific template to apply in that folder. This is the quickest method to create a new note based on a template.

Again this can be done in two ways. Either we right-click on the folder and select the command New note

Or we can press hotkey Ctrl+N from inside a currently open file.
In either way the template will be applied only if we have configured the Templater plugin with the Folder Templates option

When the Enable Folder Templates option is true templates are triggered when a new empty file is created in a give folder. Templater will try to fill the empty file with the specified template automatically.
We must also specify which template will be applied on the specific folder.
